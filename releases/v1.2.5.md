
# Release 1.2.5 (2026-06-24)

## Features

- **multi-version MQ testing via mq-versions.json manifest (#158)**
* docs(plans): design for multi-version MQ testing (#157)

Adds the approved design for supporting more than one IBM MQ version in the dev environment. A single mq-versions.yml manifest becomes the source of truth; lifecycle scripts, the setup-mq action, and a reusable workflow read it. One version runs per test run on the stable contract ports, with 10.0 as the default and 9.4.5 retained in the matrix. The environment exports the resolved version and a capability map so consuming-repo tests can make version-conditional assertions.

Scope is the mechanism in this repo only; consuming-repo adoption and version-lifecycle policy are deferred to separate cycles.
