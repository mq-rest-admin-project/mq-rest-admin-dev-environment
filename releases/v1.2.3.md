
# Release 1.2.3 (2026-06-23)

## Bug fixes

- **pass boolean to ci-security reusable workflow inputs (#125)**
The security reusable workflow declares run-standards and run-security as type: boolean. The caller was passing values via the || operator which produces strings, causing the reusable workflow to fail silently. Use the != 'false' pattern to produce actual booleans. Ref wphillipmoore/standard-tooling#632

- **pin MQ image to 9.4.5.1-r1 (latest supported 9.x) (#148)**
The queue-manager image floated on icr.io/ibm-messaging/mq:latest, which rolled to MQ 10.0.0. The fleet does not support v10 yet, and real MQ 10.0 returns new quantum-safe DISPLAY CHSTATUS attributes (SSLQS/SSLQSR) that mapping-data.json does not map, so every language repo's strict response mapper raised and the integration CI gate failed.

Pin the compose default to 9.4.5.1-r1 (highest 9.x multi-arch tag in icr.io; the only 10.x tag is 10.0.0.0-r1). The MQ_IMAGE override is preserved. Update the current-state config/reference docs to match; historical release and dated design-decision docs are intentionally left unchanged.

v10 enablement (including the SSLQS/SSLQSR mappings) is tracked in mq-rest-admin-common#217.
