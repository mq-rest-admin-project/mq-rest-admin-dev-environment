
# Release 1.2.4 (2026-06-23)

## Bug fixes

- **gate lifecycle on queue-manager stability window (#152) (#153)**
The integration suite flaked because mq_start.sh's readiness wait returned after a single successful REST probe. During startup the MQ web server answers one request while the queue manager is still stabilising, then resets connections — surfacing downstream as SSL_ERROR_SYSCALL / Connection reset by peer in the consuming repos' test steps after setup-mq had already gone green.

Extract a shared mq_wait_ready.sh gate that requires N consecutive healthy probes across both queue managers (a stability window), with curl retry hardening and an explicit timeout message that distinguishes a genuine startup failure from a slow start. mq_start.sh, mq_seed.sh, and mq_verify.sh all gate through it, so seed/verify and standalone callers inherit the same window. The setup-mq Seed step now passes the REST port env so the gate probes the correct ports.
